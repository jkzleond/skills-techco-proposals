## 快速开始

**首次使用**：AI Agent 会询问你的公司信息并保存到 `.claude/user-config.yaml`

**之后使用**：直接提供需求，例如：

> 用户：为 XX 协会编制会员积分商城建设方案

> AI：[按照规范输出完整方案]

**核心原则**：方案只回答 4 个问题
1. 为什么要做（Why）→ 项目背景与建设目标
2. 做什么（What）→ 功能方案设计
3. 要花多少钱（How much）→ 投资预算
4. 要多久（How long）→ 实施周期

**快速工具**：
```bash
# 生成方案模板
python3 presales-proposal/scripts/create_proposal_template.py "项目名称"

# 验证方案
python3 presales-proposal/scripts/validate_proposal.py proposal.md

# 检查 ASCII 图标注
python3 converting-markdown/scripts/check_ascii_blocks.py proposal.md
```

## AI 交互流程

### 步骤 1：检查用户配置

**读取配置**：`.claude/user-config.yaml`

- ✅ 配置存在 → 使用 `company_name`，跳过询问
- ❌ 配置不存在 → 询问用户信息：
  1. 您的姓名？（可选）
  2. 您的公司名称？
  3. 您在公司担任什么角色？

**保存格式**：
```yaml
user:
  name: "张三"
  company_name: "某某科技有限公司"
  company_role: "售前产品经理"

last_updated: "2026-02-08"
```

### 步骤 2：编制方案

使用配置中的 `company_name` 作为编制单位，按照 SKILL 规范编制方案。

### 步骤 3：输出方案

确保文档头部信息正确：
```markdown
# [项目名称]建设方案

**编制单位：** [从配置读取的公司名称]
**编制日期：** 2026年X月
**版本号：** v1.0
```

## 核心工作原则

### 方案只回答四个核心问题

1. **为什么要做**（Why）- 项目背景与建设目标
2. **做什么**（What）- 功能方案设计
3. **要花多少钱**（How much）- 投资预算
4. **要多久**（How long）- 实施周期

### 去除所有不必要的内容

**❌ 不要写**：
- 技术架构设计、数据库设计、接口设计
- 代码实现细节、技术选型对比
- 投入产出分析、投资回收期计算
- 价值量化数据、案例参考
- 售后服务、质保期、维护计划
- 附录（技术说明、风险提示、商务条款）
- 具体商品清单/数据清单

### 文档风格要求

**✅ 要做到**：
- 段落式叙述，逻辑连贯
- 图文并茂（表格、流程图、ASCII 图）
- 使用引用、示例增强说服力

**❌ 要避免**：
- PPT 式要点清单
- 过多的无序列表
- 枯燥的技术术语

## 方案结构要求

### 一、项目背景与建设目标（二级标题 ##）

**包含内容**：
- 现状分析（当前情况、存在的问题）
- 会员/客户的真实反馈（使用引用增强说服力）
- 解决方案思路（用流程图展示，⭐ **必须**使用 ` ```ascii:flowchart ` `）
- 建设目标（用表格呈现）

**⚠️ ASCII 图规范**：所有图表必须标注类型，详见 `references/ascii-art-specs.md`

**标题示例**：
- ✅ "## 一、项目背景与建设目标"
- ❌ "## 一、为什么要做"

### 二、功能方案设计（二级标题 ##）

**包含内容**：
- 系统定位（是什么、不是什么）
- 系统架构（用 ASCII 图展示，⭐ **必须**使用 ` ```ascii:architecture ` `）
- 功能设计（图文并茂，用表格、示例、流程图）
- 与现有系统的关系（用架构图说明，⭐ **必须**使用 ` ```ascii:architecture ` `）

**⚠️ ASCII 图规范**：所有图表必须标注类型，详见 `references/ascii-art-specs.md`

**三级标题示例**：
- ✅ "### 2.1 系统架构：不折腾，简单好用"
- ✅ "### 2.2 会员端功能：简单直观"
- ✅ "### 2.3 管理端功能：自主可控"

### 三、投资预算（二级标题 ##）

**包含内容**：
- 开发费用（明确数字，中文大写）
- 费用构成（用表格展示功能模块，让客户看到价值）
- 费用包含内容（按阶段展开工作量）
- 费用不包含内容（表格形式）
- 付款方式（用流程图展示，⭐ **必须**使用 ` ```ascii:flowchart ` `）

**推荐方式**：混合拆分（模块 + 阶段）
- 先展示功能模块表格，让客户看到"钱花在哪些功能上"
- 再展开开发阶段，让客户了解"我们会做这么多事"

### 四、实施周期（二级标题 ##）

**包含内容**：
- 开发周期（时间估算）
- 实施阶段（分阶段详细说明）
- 交付物清单（分类列出）

**❌ 不要写**：
- 售后服务、质保期、维护计划

## 场景变体

### 小型项目（预算 < 10万）

- 简化功能设计章节
- 投资预算采用简单表格
- 实施周期按整体说明

### 大型项目（预算 > 50万）

- 增加分阶段实施方案
- 投资预算增加详细说明
- 实施周期增加里程碑节点

### 紧急项目（周期 < 1个月）

- 精简文档，聚焦核心功能
- 实施周期按周说明
- 交付物清单列出核心项
